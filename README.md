# Life Roadmap

Личное веб-приложение для глубокой саморефлексии и психологического самоанализа. Визуальная карта жизненного пути с научно обоснованными метриками психологического здоровья.

## Идея

Большинство инструментов трекинга жизни работают на уровне привычек или задач. Этот проект идёт глубже — он структурирует **события жизни** как данные и анализирует их через призму доказательной психологии: от пирамиды Маслоу до теории ментализации Фонаги.

Цель: видеть не просто хронологию, а **паттерны** — какие события запускают цепочки других, как менялась психологическая устойчивость, где дефициты и где ресурсы.

Проект полностью **local-first** — данные остаются у пользователя.

## Функциональность (текущий фронтенд)

### Таймлайн
- Горизонтальная хроника жизни с карточками событий
- **8 UX-режимов:**
  - `Impact Mapping` — смещение карточек выше/ниже оси по значимости события
  - `Temporal Chromatics` — насыщенность цвета по эпохам (приглушённое прошлое / яркое настоящее)
  - `Compact Mode` — свёрнутый вид карточек
  - `Causal Lineage` — SVG-линии причинно-следственных связей между событиями
  - `Trend Overlay` — наложение кривых энергии и стресса поверх таймлайна
  - `Semantic Zoom` — три уровня детализации: события / эпохи / декады
  - `Progressive Disclosure` — боковая панель с полным психологическим профилем события
  - `Navigation Minimap` — фиксированная миникарта с синхронизацией прокрутки

## Что предстоит

Подробный план с чекбоксами — в [`ROADMAP.md`](./ROADMAP.md).

### Фаза 2 — Бэкенд
- PostgreSQL 16 + расширение `pgvector` для семантического поиска по записям
- Схема "Событие → Реакция" (`parent_id → child_id`) для цепочек любой вложенности
- JSONB-поля для психологических метрик — добавление новых шкал без миграций
- FastAPI + Pydantic v2 — строгая валидация всех входящих данных
- Alembic-миграции, Recursive CTE для сборки цепочек событий
- Materialized Views для Маслоу / Колеса жизни (обновление по триггеру, не при каждом запросе)
- Audit Log через Postgres-триггеры — undo/redo без логики в API
- Ежемесячные снапшоты психологического состояния через ARQ-воркер
- Автобэкап: `pg_dump` → зашифрованный локальный архив

### Фаза 3 — Интеграция
- Замена статических данных (`src/data/`) на `fetch('/api/v1/...')`
- Создание/редактирование событий через Drawer → POST/PUT
- Семантический поиск через pgvector
- Undo через историю из audit_log
- Service Worker для офлайн-режима

### Фаза 4 — Упаковка
- Docker Compose — один `docker compose up` запускает всё
- `.desktop`-файл для запуска из меню Linux
- Tauri-обёртка для десктоп-приложения без браузера (опционально)
